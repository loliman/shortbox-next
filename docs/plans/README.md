# Implementation Plans

This folder contains **Implementation Plans**.

While a specification (`docs/specs`) describes *what* to build, an implementation plan describes *how* to build it. It translates a feature specification into concrete, actionable steps and file-level changes.

## Purpose

- Clearly map out required file additions, deletions, and modifications.
- Define test plans and manual verification steps prior to execution.
- Validate architectural placement (`app/`, `src/services/`, etc.) before code is written.
- Provide a structured checklist for AI Agents and Developers to ensure reliable execution.

## Usage

Use the [`implementation-plan-template.md`](./implementation-plan-template.md) when preparing for a complex implementation.
For simple or one-off modifications, a formal plan is usually not needed.
